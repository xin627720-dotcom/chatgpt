# 高考英语词汇学习 App：Codex 开发需求文档 v0.1

## 0. 一句话目标

做一个面向高中生/高考生的英语词汇学习 App。

核心思想：

英语做题技巧、作文技巧、考试大招，都要建立在词汇基础上。  
所以 App 要先帮助学生把高考词汇背熟、复习稳、测得准，再逐步扩展到做题和作文。

---

## 1. 资料来源

### 1.1 词汇资料

已有两份词汇 PDF：

1. `01 正序极速文字版FREE高考英语2050核心词汇表.pdf`
   - 用途：作为主词库。
   - 内容形态：英文单词 + 中文释义，按字母正序排列。
   - 示例：`abandon 抛弃、放弃`、`ability 能力`、`absorb 吸收、理解`、`academic 学术的` 等。

2. `02 正序极速文字版FREE高考英语700基础词汇表.pdf`
   - 用途：作为基础词库。
   - 内容形态：英文单词 + 中文释义，按字母正序排列。
   - 示例：`a (an) 一个、某个`、`afternoon 下午`、`apartment 公寓`、`button 纽扣、按钮` 等。

### 1.2 视频资料

上传的视频片段标题为“词汇应该怎么背”。

视频片段中老师表达的核心观点是：

- 英语考试里的题可以教方法；
- 作文也可以教写法；
- 所谓技巧和大招也有；
- 但所有这些都依赖一件事：词汇。

本 App 的产品设计要围绕这句话展开：

**不是做一个花哨的背词软件，而是做一个以“词汇为考试能力底座”的学习系统。**

## 2. 产品定位

### 2.1 产品名称

暂定：`高考英语词汇底座`

也可后续改名为：

- 高考词汇通
- 词汇底座
- 高考英语 2750 词
- 高考英语词汇冲刺

### 2.2 目标用户

1. 高一到高三学生。
2. 高考英语基础薄弱、单词量不足的学生。
3. 已经学过语法和技巧，但阅读、完形、作文仍然卡在词汇上的学生。
4. 老师或家长可以用来监督学生背词。

### 2.3 核心价值

用户打开 App 后，不只是“看单词表”，而是完成一套闭环：

**看词 → 回忆 → 判断会不会 → 错词沉淀 → 定时复习 → 小测验 → 看到进度**

## 3. MVP 第一版功能范围

第一版先做最重要的，不做复杂社交、不做支付、不做 AI 大模型生成内容。

### 3.1 首页 Dashboard

首页显示：

- 今日应学单词数
- 今日应复习单词数
- 当前连续学习天数
- 700 基础词掌握进度
- 2050 核心词掌握进度
- 快捷入口：开始学习、开始复习、搜索单词、错词本

### 3.2 词库模块

系统内置两套词库：

1. 700 基础词库
2. 2050 核心词库

每个单词至少包含：

```json
{
  "id": "word_abandon_core_001",
  "word": "abandon",
  "meaning_cn": "抛弃、放弃",
  "book": "core_2050",
  "order_index": 1,
  "letter": "A"
}
```

字段说明：

- `id`：唯一编号。
- `word`：英文单词。
- `meaning_cn`：中文释义。
- `book`：所属词库，`basic_700` 或 `core_2050`。
- `order_index`：在原始词表中的顺序。
- `letter`：首字母分类。

### 3.3 单词列表页

功能：

- 可切换 700 基础词 / 2050 核心词。
- 按 A-Z 分组显示。
- 支持搜索英文或中文。
- 每个词显示：英文、中文释义、掌握状态。
- 点击进入单词详情页。

掌握状态分为：

- 未学
- 认识
- 模糊
- 已掌握

### 3.4 单词卡片学习模式

这是 App 的核心页面。

学习流程：

1. 显示英文单词。
2. 用户先在心里想中文意思。
3. 点击“显示答案”。
4. 显示中文释义。
5. 用户选择：
   - 认识
   - 模糊
   - 不认识
6. 系统根据选择更新复习时间和掌握状态。

页面按钮：

- 显示答案
- 认识
- 模糊
- 不认识
- 收藏
- 下一个

### 3.5 复习算法 MVP

第一版不需要复杂算法，先用简单规则：

- 选择“不认识”：加入错词本，10 分钟后可再次复习，明天继续出现。
- 选择“模糊”：当天学习结束后再出现一次，2 天后复习。
- 选择“认识”：3 天后复习。
- 连续 3 次选择“认识”：标记为“已掌握”。

数据字段建议：

```json
{
  "word_id": "word_abandon_core_001",
  "status": "learning",
  "last_reviewed_at": "2026-05-11T00:00:00Z",
  "next_review_at": "2026-05-14T00:00:00Z",
  "correct_count": 2,
  "wrong_count": 1,
  "fuzzy_count": 1,
  "is_favorite": false
}
```

### 3.6 错词本

错词本自动收集：

- 用户点“不认识”的词
- 测验答错的词
- 用户手动收藏/标记的词

错词本功能：

- 列表查看
- 重新学习
- 只测错词
- 从错词本移除

### 3.7 测验模式

第一版做两种测验：

#### A. 英译中选择题

显示英文单词，给 4 个中文选项。

#### B. 中译英选择题

显示中文释义，给 4 个英文选项。

答题后：

- 立即显示对错。
- 错题自动加入错词本。
- 记录测验得分。

### 3.8 学习计划

用户可以设置：

- 每天学习新词数量：10 / 20 / 30 / 50
- 先学哪个词库：700 基础词优先 / 2050 核心词优先
- 是否每日提醒

默认建议：

- 基础弱：先学 700 基础词，每天 20 个。
- 基础一般：直接学 2050 核心词，每天 30 个。

### 3.9 视频学习方法页

该页面不播放原视频，第一版只放整理后的学习理念。

标题：

`为什么词汇是英语考试的底座？`

内容：

- 做题技巧有用，但看不懂单词，技巧发挥不出来。
- 作文模板有用，但没有词汇，句子写不出来。
- 阅读、完形、七选五、作文都依赖词汇。
- 所以 App 的学习顺序是：先背核心词，再反复测，再处理错词。

## 5. 数据模型

### 5.1 Word

```ts
export type WordBook = 'basic_700' | 'core_2050';

export interface Word {
  id: string;
  word: string;
  meaningCn: string;
  book: WordBook;
  letter: string;
  orderIndex: number;
}
```

### 5.2 UserWordProgress

```ts
export type WordStatus = 'new' | 'learning' | 'fuzzy' | 'mastered';

export interface UserWordProgress {
  wordId: string;
  status: WordStatus;
  lastReviewedAt?: string;
  nextReviewAt?: string;
  correctCount: number;
  wrongCount: number;
  fuzzyCount: number;
  isFavorite: boolean;
}
```

### 5.3 QuizResult

```ts
export interface QuizResult {
  id: string;
  createdAt: string;
  book: WordBook | 'wrong_words' | 'mixed';
  total: number;
  correct: number;
  wrong: number;
  score: number;
}
```

### 5.4 UserSettings

```ts
export interface UserSettings {
  dailyNewWords: number;
  preferredBook: WordBook;
  reminderEnabled: boolean;
}
```

## 6. 技术建议

第一版建议做 Web App，手机浏览器也能用，后续再封装成 App。

推荐技术栈：

- Next.js
- TypeScript
- Tailwind CSS
- 本地存储：localStorage 或 IndexedDB
- 后续可升级：Supabase 登录和云同步

第一版可以不做登录，先把数据保存在本机。这样最快做出可用版本。

## 8. 开发任务拆分

### Phase 1：项目搭建

- 创建 Next.js + TypeScript 项目。
- 配置 Tailwind CSS。
- 建立页面路由。
- 建立基础布局和底部导航。

### Phase 2：词库数据

- 把 PDF 词汇表整理成两个 JSON 文件。
- 实现词库加载。
- 实现搜索和 A-Z 分类。

### Phase 3：学习模式

- 实现单词卡片。
- 实现“显示答案”。
- 实现“认识/模糊/不认识”。
- 实现学习进度保存。

### Phase 4：复习和错词本

- 实现简单复习时间规则。
- 实现错词本列表。
- 实现错词复习。

### Phase 5：测验模式

- 实现英译中选择题。
- 实现中译英选择题。
- 实现错题记录和测验结果。

### Phase 6：打磨

- 手机端适配。
- 空状态提示。
- 简单动画。
- 数据重置功能。

## 9. 验收标准

第一版完成后，必须满足：

1. 用户能选择 700 基础词或 2050 核心词。
2. 用户能搜索单词。
3. 用户能用卡片模式背单词。
4. 用户能标记认识、模糊、不认识。
5. 不认识的单词会进入错词本。
6. 用户能做选择题测验。
7. 答错的词会进入错词本。
8. 首页能显示学习进度。
9. 刷新页面后，学习记录不丢失。
10. 手机屏幕上使用舒服。
